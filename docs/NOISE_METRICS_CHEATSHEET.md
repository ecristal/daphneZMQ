# DAPHNE Noise Metrics Cheat Sheet

This sheet explains the metrics generated by:
- `client/analyze_waveform_dataset.py`

## 1) Signal Interpretation Basics
- `mean_counts`: DC baseline level.
- `median_counts`: robust baseline (less sensitive to spikes).
- `std_counts`: overall spread around the mean.
- `ac_rms_counts`: RMS after removing DC; usually the best single "noise amplitude" value.
- `rms_counts`: includes DC and AC; dominated by offset if baseline is large.
- `min_counts`, `max_counts`, `p2p_global_counts`: extrema and full span over the whole run.

## 2) Robust Distribution Metrics
- `mad_counts`: Median Absolute Deviation, robust spread estimate.
- `p01_counts`, `p05_counts`, `p95_counts`, `p99_counts`: tails of the amplitude distribution.

How to read:
- Large gap between percentile tails and Gaussian expectation can indicate non-Gaussian noise, clipping, or bursts.

## 3) Per-Waveform Stability Metrics
- `wf_mean_mean_counts`: mean of each waveform mean (baseline center).
- `wf_mean_std_counts`: baseline drift over time.
- `wf_std_mean_counts`: average in-waveform noise level.
- `wf_std_std_counts`: stability of that noise level across waveforms.
- `wf_p2p_mean_counts`: average waveform p2p.
- `wf_p2p_std_counts`: variability of waveform p2p.

How to read:
- High `wf_mean_std_counts` suggests slow baseline motion.
- High `wf_p2p_std_counts` suggests intermittent disturbances.

## 4) Shape Metrics
- `skewness`: asymmetry of sample distribution.
  - Near `0`: symmetric.
  - Positive: heavier high-side tail.
  - Negative: heavier low-side tail.
- `kurtosis_excess`: tail heaviness relative to Gaussian.
  - Near `0`: Gaussian-like.
  - Positive: impulsive/heavy-tailed behavior.
- `crest_factor`: `(0.5 * p2p_global) / ac_rms`.
  - Large values indicate peaky/impulsive events.

## 5) FFT Metrics (Power-Averaged)
- `fft_peak_freq_hz`: strongest spectral bin frequency.
- `fft_peak_dbfs`: strongest bin level in dBFS/bin.
- `fft_floor_median_dbfs`: median floor of FFT bins (excluding peak neighborhood).
- `fft_floor_mean_dbfs`: mean floor of FFT bins.
- `fft_peak_minus_floor_db`: peak-to-floor separation.
- `fft_avg_waves_used`: number of waveforms included in FFT averaging.

How to read:
- Higher `fft_floor_median_dbfs` (less negative) = noisier broadband floor.
- Large `fft_peak_minus_floor_db` = clear tone/spur above floor.

## 6) Outlier Metrics
Outlier detection uses robust z-score across channels for:
- `std_counts`
- `wf_p2p_mean_counts`
- `fft_floor_median_dbfs`

Reported fields:
- `std_counts_robust_z`
- `wf_p2p_mean_counts_robust_z`
- `fft_floor_median_dbfs_robust_z`
- `outlier_score_max_abs_z`
- `outlier_flag`
- `outlier_reasons`

Default threshold:
- `|robust z| >= 3.5` flags a channel.

## 7) Practical Workflow
1. Start with `ac_rms_counts`, `std_counts`, and `fft_floor_median_dbfs`.
2. Check `wf_mean_std_counts` for baseline drift.
3. Check `wf_p2p_std_counts`, `kurtosis_excess`, `crest_factor` for bursty/impulsive behavior.
4. Inspect `fft_peak_freq_hz` and `fft_peak_minus_floor_db` for coherent spurs.
5. Review `outlier_flag` and `outlier_reasons` to focus channel-by-channel debugging.

## 8) Notes
- Units:
  - Time-domain metrics are in ADC counts.
  - Frequency-domain levels are in dBFS/bin.
- `ac_rms_counts` and `std_counts` should be very close for stable signals.
- If interpreting files as signed ADC data, use `--interpret signed` (default).
